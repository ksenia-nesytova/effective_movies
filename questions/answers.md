**Even Propagataion** - порядок, согласно которому событие движется по DOM.

_**Even Propagataion**_ состоит из **3 основных фаз**:
* **Capturing** (Перехват) - событие проходит путь сверху DOM(порядок обычно примерно следующий: window > document > html > body > ... > target element) до цели(event target).
* **Target** - Событие достигает цели(event target) (формально можно не выносить в отдельную фазу). Обработчики, 
* **Bubbling** (Всплытие) - событие двигает от event target к верху DOM(до window). Это дефолтное направление движения события. Здесь отрабатывают обработчики, назначенные на even target. 

При регистрации обработчика можно изменить направление движения события на Перехват.

**Что со всем этим делать**

Обработчик событий отрабатывает только в указанном контексте.
* Так, если мы полагаемся на Всплытие и хотим обработать одно и то же событие, например, click, для контейнера и его дочерних элементов, мы можем назначить обработчик только для родительского контейнера, и при клике на любой из дочерних элементов, событие "всплывет" до элемента, на которое назначен обработчик, и будет обработано корректно.

```html
//При клике на любой из параграфов внутри section(event target), событие будет обработано обработчиком события родительского элемента. Поскольку мы передаем объект $event, то мы можем узнать, на который из параграфов нажали
<section (click)="doSomething($event)">
<p>Click me!</p>
<p>Or me!</p>
</section>
```

* Также Bubbling позволяет контролировать, на каком элементе событие остановится. Например, у нас есть карточка, нажатие на которую перенаправляет пользователя на другую страницу. Внутри это карточки есть интерактивные элементы(dropdown, radio btn, etc). Чтобы опльзователь мог взаимодействовать с подобными элементами без того, чтобы его перебрасывало на новую страницу, можно остановить распространение событий нв интерактивном элементе(_event.stopPropagation()_)

* Возможен обратный случай, когда нам нужно сделать так, чтобы событие **не** достигло event target. Например, можно кликать на любой элемент страницы и будет произведено какое-то действие(например, вызов модального окна). 
```html
//Если при добалвении обрабочика события добавить флаг _capture: true_, то мы можем остановить событие до того, ка коно достигнет последнего параграфа
<section (click)="showModal()">
<p>Click me!</p>
<p>Or me!</p>

<p id="copiable">Copy me without seeing pop-ups</p>
</section>
```
